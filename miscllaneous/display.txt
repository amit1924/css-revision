Block-Level Elements
Block-level elements typically start on a new line and take up the full width available. They are used for structuring the main parts of a document.

<div>: A generic container for content.
<h1> to <h6>: Headings.
<p>: Paragraph.
<ul>, <ol>, <li>: Lists and list items.
<table>, <thead>, <tbody>, <tr>, <td>: Table elements.
<header>, <footer>, <section>, <article>, <aside>, <nav>: Semantic HTML5 elements for defining different sections of a document.
<form>: Form container.
<blockquote>: Block quotation.
<figure>, <figcaption>: Figures with optional caption.

################################################################################################################################
Inline-Level Elements
Inline-level elements do not start on a new line and only take up as much width as necessary. They are used for formatting content within block-level elements.

<span>: A generic container for inline content.
<a>: Anchor/link.
<img>: Image.
<strong>, <b>: Bold text.
<em>, <i>: Italic text.
<code>, <pre>: Code and preformatted text.
<small>: Small text.
<sup>, <sub>: Superscript and subscript.
<br>: Line break.
<label>: Label for form elements.
<input>, <button>, <select>, <textarea>: Form elements.
